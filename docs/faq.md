# FAQ — Loki 200mm ROS2 Robot

## Общие вопросы

### Что такое Loki 200mm?
MakersPet Loki — это open-source мобильный робот диаметром 200мм на дифференциальном приводе. Платформа разработана компанией [MakersPet](https://github.com/makerspet) и предназначена для обучения ROS2 и навигации.

### Чем этот проект отличается от оригинального makerspet_loki?
Данный проект добавляет:
- Поддержку microROS (ESP32 как low-level контроллер вместо Arduino Mega)
- Расширенную поддержку LiDAR через библиотеку kaiaai/LDS (20+ моделей)
- Датчики тока INA219 для мониторинга батареи
- OTA обновления прошивки
- Улучшенные launch-файлы для Cartographer SLAM и Nav2

### Нужен ли ROS2 для работы?
Да. ESP32 работает автономно (ПИД, энкодеры, LiDAR), но для SLAM и навигации необходим Raspberry Pi 4 с ROS2 Iron.

## Железо

### Какой LiDAR рекомендуется?
**YDLidar X4** — оптимальный выбор: 0.12–10м, 5000 сэмплов/сек, 6–12Гц, ~$60. Полный список совместимых LiDAR в [docs/lidar_compatibility.md](lidar_compatibility.md).

### Можно ли использовать другой мотор-драйвер вместо L298N?
Да. Прошивка через kaiaai/firmware поддерживает: TB6612FNG, DRV8871, DRV8833, DRV8835. Для BLDC — JGA25-BL2418. Рекомендуется TB6612FNG как более эффективная альтернатива L298N.

### Какой аккумулятор использовать?
LiPo 3S (11.1В): рекомендуется ёмкость 2200–3000мАч. Диапазон рабочего напряжения: 10.5В (разряд) — 12.6В (полный). Встроенный мониторинг через ADC (GPIO36) + INA219.

### Сколько весит робот?
Около 1.52кг в полной сборке (корпус + RPi4 + ESP32 + LiDAR + аккумулятор).

### Какой ток потребляет система?
- Raspberry Pi 4 (4GB): ~600–800мА при нагрузке
- ESP32: ~200–300мА
- LiDAR (YDLidar X4): ~300мА
- Моторы: до 1А каждый при движении
- **Итого**: 3–4А от батареи. Используйте buck-конвертер на 5А.

### Где купить детали?
- ESP32 DevKit v1: AliExpress ~$4
- L298N: AliExpress ~$2
- BNO055: AliExpress ~$5
- INA219: AliExpress ~$2 (2 шт.)
- JGA25-370 (12В 40:1): AliExpress ~$8/шт
- YDLidar X4: YDLidar store / AliExpress ~$60
- Raspberry Pi 4 (4GB): официальные дилеры ~$55
- 3D модели для печати: [MakersPet GitHub](https://github.com/makerspet)

## Прошивка (ESP32)

### Какой WiFi нужен?
Только 2.4GHz. ESP32 не поддерживает 5GHz.

### Можно ли обновлять прошивку по воздуху?
Да, через OTA (PlatformIO: `pio run --target upload --upload-port loki.local`).

### Что такое microROS?
micro_ros_arduino/micro_ros_kaia — реализация ROS2 для микроконтроллеров. ESP32 выступает ROS2 нодой, публикуя `/odom`, `/scan`, `/imu/data`, `/battery_state` и подписываясь на `/cmd_vel`.

### Как ESP32 связывается с RPi?
Через UDP на порт 8888. ESP32 отправляет данные на IP-адрес RPi, где запущен `micro_ros_agent`.

### Какова частота обновления данных?
- Одометрия (`/odom`): 50Гц
- LiDAR (`/scan`): 6–12Гц (зависит от модели)
- IMU (`/imu/data`): 100Гц
- Батарея (`/battery_state`): 1Гц

## ROS2 / Программное обеспечение

### Какие версии ROS2 поддерживаются?
**ROS2 Iron** (Ubuntu 22.04 LTS). Также должно работать с Humble, но не тестировалось.

### Почему Cartographer, а не SLAM Toolbox?
Cartographer показывает лучшие результаты с LiDAR без IMU (у Loki IMU используется только для EKF, не для SLAM). SLAM Toolbox работает, но требует дополнительной настройки.

### Какова максимальная скорость робота?
- Максимум прошивки: 0.61 м/с линейная, 2.84 рад/с угловая
- Рекомендуемая для SLAM: ≤0.26 м/с линейная (настройка Nav2)

### Как запустить автономное исследование помещения?
```bash
# Терминал 1: запуск всего стека
ros2 launch loki_robot robot.launch.py

# Терминал 2: автономное исследование
ros2 launch loki_robot explore.launch.py
```
Робот автоматически исследует помещение и вернётся в начальную точку.

### Сколько памяти нужно на RPi для карты?
Небольшая квартира (50м²): ~50МБ RAM. Cartographer держит карту в памяти. После завершения сохраните: `ros2 run nav2_map_server map_saver_cli -f my_map`.

### Как сохранить и загрузить карту?
```bash
# Сохранить
ros2 run nav2_map_server map_saver_cli -f ~/maps/my_room

# Загрузить (добавьте в launch-файл)
ros2 run nav2_map_server map_server --ros-args -p yaml_filename:=my_room.yaml
```

## 3D печать

### Из чего печатать?
- Структурные детали (основание, моторные кронштейны): **PETG** — прочность + устойчивость к теплу
- Декоративные детали (голова): **PLA** — легче печатать

### Сколько времени занимает печать?
Примерно 20–30 часов для полного комплекта деталей на принтере с соплом 0.4мм.

### Поддержки нужны?
Большинство деталей спроектированы без необходимости поддержек. Детали с отверстиями под подшипники могут требовать поддержек.

## Разработка

### Как добавить новый сенсор?
1. Подключить к ESP32 (I²C/UART/GPIO)
2. Добавить код в `firmware/src/sensors.cpp`
3. Создать ROS2 publisher в `firmware/src/microros_bridge.h`
4. Добавить визуализацию в RViz config

### Как запустить симуляцию в Gazebo?
```bash
ros2 launch loki_robot gazebo.launch.py
```
Модель загружается из `ros2/sdf/makerspet_loki/model.sdf`.

### Можно ли запустить на другой платформе (не Raspberry Pi)?
Да, любой компьютер с Ubuntu 22.04 и ROS2 Iron подойдёт. Для реального робота нужен только `micro_ros_agent`.
